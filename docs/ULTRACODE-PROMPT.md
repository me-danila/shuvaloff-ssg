```

Задача: автономно реализовать ТОЛЬКО автономные пункты из docs/AUDIT-PLAN.md для academia-shuvaloff.ru, оркестрируя субагентов через Workflow, и довести до полностью готового, верифицированного результата.

ПЕРЕД СТАРТОМ прочитай: docs/AUDIT-PLAN.md (полностью) и память проекта (MEMORY.md + audit-plan-constraints + v2-rollout-strategy). docs/AUDIT-PLAN.md — единственный источник правды по объёму работ.

ЖЁСТКИЕ ОГРАНИЧЕНИЯ (нарушение = откат):
1. Только улучшать. Никакой деградации UI/UX, ноль риска сломать сторонние виджеты.

5. Только автономные пункты. НЕ трогать раздел «Требуется от отеля» (цены, рейтинги, sameAs-URL, данные ресторана, bio автора, эндпоинт рассылки, подтверждения дизайн-намерения). Если пункт внутри фазы требует данных/эндпоинта/подтверждения — пропустить и зафиксировать в финальном отчёте.

ОБЪЁМ (автономные фазы плана, в рекомендованном порядке): Фаза 0 → 1 → 2 → 3 → 4 → 5 → 9 → 6 → 8 → 7. Точные пункты/файлы/строки — в docs/AUDIT-PLAN.md.

ОРКЕСТРАЦИЯ (Workflow, по одной фазе за проход, ты в цикле между фазами):
- Ветка: создать feat/audit-autonomous от main. Коммит per-фаза, осмысленные сообщения. НЕ пушить без спроса.
- Каждая фаза = отдельный workflow: (a) scout — уточнить точные файлы/строки и зависимости; (b) implement — независимые пункты параллельно (isolation:'worktree' там, где правки одних файлов конфликтуют); (c) verify — адверсариально: сборка + lint + DOM-проверка + «дизайн не изменился».
- Зависимости соблюдать: Фаза 1 (снять forceReload) ДО Фазы 8 (<a>→next/link). Фаза 2 #1 (locale-проп) может задевать много компонентов — делать инкрементально.
- После каждой фазы читай результат verify перед следующей.

DONE-КРИТЕРИЙ каждой фазы (всё должно пройти):
- bun lint (biome) — чисто.
- bun run build (next build && next-image-export-optimizer) — успешно, без новых ворнингов.
- Preview: поднять dev-сервер, проверить структуру/computed-styles ключевых затронутых экранов. ВАЖНО: remote-картинки с academia.spb.ru в preview не грузятся — верифицировать DOM/getComputedStyle, НЕ скриншоты.
- Дизайн не изменился: сравнить computed-styles/DOM затронутых компонентов до/после.
- Для SEO/schema (Фазы 4,5,9): JSON-LD валиден и парсится, @id связаны; метатеги/hreflang/robots присутствуют в сгенерированных out/**/*.html.

ГОЧИ:
- Tailwind v4 кэширует @theme в .next → после правок app/globals.css делать rm -rf .next перед проверкой.

ФИНАЛ: отчёт по фазам — что сделано, что пропущено и почему (со ссылкой на «Требуется от отеля»), diff-стат, статус build/lint, список коммитов. Отметить выполненные пункты в docs/AUDIT-PLAN.md.
```
